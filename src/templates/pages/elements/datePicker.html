{% extends 'layouts/base.html' %} {% block main %} {% markdown %} ## Date Picker Use this element to provide an easy and comfortable way to enter dates into input fields. It was **designed for use on desktop browsers only**. On mobile browsers it makes more sense to use the native input widgets. | | | | - | - | | Less-File | [datePicker.less](assets/less/elements/datePicker.less) | | JS-File | [datePicker.js](assets/js/modules/datePicker.js) | | Base CSS-Class | `.datePicker` | | Modifiers | `none` | ### Basic Example Add the data-attribute "datepicker" to any (date-)input field that you would like to use the date picker. A little icon on the right side will appear to indicate that the field invokes a date picker on focus. *If you do not supply any more data, the date picker will start with the current date*. {% endmarkdown %}
{% markdown %} ### Set an Initial Date You can add a date to the input field via markup. **Make sure you use the date picker data-attribute. Do not use the value-attribute.** {% endmarkdown %}
{% markdown %} If you leave out a parameter, it will **fall back to the current date**. Eg. if you leave out the year, the date picker will assume you wish to initialize it with the current year. Same for month and day, of course. {% endmarkdown %}
{% markdown %} ### Positioning By default, the date picker appears below the date input field. However, if the available space in viewport below the input is too low, the date picker appears above the input field. Try it out by scrolling the page before you focus on the input field. {% endmarkdown %}
{% markdown %} ### Large Input Fields The date picker works perfectly fine with both large and small input fields. {% endmarkdown %}
