{% extends 'layouts/base.html' %} {% block main %} {% markdown %} ## MicroSubmit This script provides a simple interface to post a string to a given endpoint (eg. some backend script). | | | | - | - | | Js-File | [microSubmit.js]({{ pathToSourceFile }}assets/js/modules/microSubmit.js) | | Hooks | `data-microsubmit` | {% endmarkdown %}

This script was designed for very simple use cases only! It asumes, that to the user it does not really matter if the post was succesful or not. There will always be a positive response message, if if the script failed to post.

{% markdown %} ### Simple Example To use this script, create a basic `
`-element. Define an endpoint (= backend script) via the `action`-attribute. Make sure you also add a single hidden input field which you can name however you wish. This field’s value gets posted to the endpoint. Please make sure to include a single `submit-` or `button`-element as well. *There will always be a standard, positive response*, no matter if the post fails or succeeds. {% endmarkdown %}
{% markdown %} ### Custom Response Message To add custom response message/markup, create any element you like and assign a unique `id` – eg. *myResponse* to it. Tell the script which custom response message it should use by adding `response:#myId` to the custom `data-microsubmit` attribute. Your custom response markup gets hidden on page load and revealed after the form got submitted. {% endmarkdown %}
Thank you!

Check List:

  • The DOM element which is tagged with a data-microsubmit attribute must be a form element!
  • The very form must contain a single submit- or button-element!
  • The very form must contain a single hidden input-element to define the message the script submits!
